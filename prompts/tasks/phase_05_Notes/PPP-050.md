## 2. Objective

> Create Notes, Categories, and NoteCategories tables with EF Core migration for the rich-text notes module.

This task defines the schema for the Notes module — a knowledge capture system with rich-text content, categorical organization, and optional project linking. The Notes table stores title and HTML/JSON content (from the rich text editor), with FK to Users (owner) and optional FK to Projects. Categories are user-owned (personal taxonomy). NoteCategories is a many-to-many join table. This task teaches content storage strategies (HTML vs JSON), many-to-many self-referential patterns, and schema design for searchability.

## 4. Technical Guidance

**Conventions for this module:**
- Entities: `Note`, `Category`, `NoteCategory`
- Note fields: `Id (int)`, `Title (string, max 300)`, `Content (string, nvarchar(max) — stores TipTap JSON or sanitized HTML)`, `UserId (int, FK)`, `ProjectId? (int, FK)`, `CreatedAt`, `UpdatedAt (UTC)`
- Category fields: `Id (int)`, `Name (string, max 100)`, `UserId (int, FK)`
- NoteCategory: `NoteId (int, FK)`, `CategoryId (int, FK)`, composite PK
- Index on `Notes.Title` for search; index on `Notes.UserId`
- Table names: `Notes`, `Categories`, `NoteCategories`

**Concepts practiced in this task:**
- Content storage strategies (JSON vs HTML)
- Many-to-many relationship with join table
- User-scoped categorization (personal taxonomy)
