## 2. Objective

> Implement CreateNote command with content sanitization and `POST /api/v1/notes` endpoint.

The CreateNote handler accepts rich-text content (HTML from TipTap or JSON) and sanitizes it server-side before storage to prevent XSS. Sanitization strips `<script>` tags, `on*` event handlers, and dangerous attributes while preserving formatting tags and links. FluentValidation ensures title is required and content is not empty. The note is associated with the authenticated user and optionally a project and categories. This task teaches content sanitization, XSS prevention, and safe storage of user-generated rich content.

## 4. Technical Guidance

**Conventions for this module:**
- Command: `CreateNoteCommand` with `Title`, `Content (HTML/JSON string)`, `ProjectId?`, `CategoryIds?`
- Validator: Title required 1-300, Content required and not empty
- Handler: sanitize content using `HtmlSanitizer` library (or regex-based approach for learning — strip script/event handler/iframe tags, keep p/h1-h6/ul/ol/li/strong/em/a); create Note entity; if projectId provided, verify user is project member; attach categories if provided; save
- Content sanitizer: implement as `IContentSanitizer` in Infrastructure, used by handler
- Endpoint: `POST /api/v1/notes` with `[Authorize]`
- Response: 201 with `NoteDto` including categories and project info

**Concepts practiced in this task:**
- XSS prevention via content sanitization
- Safe storage of user-generated HTML
- Optional related entity attachment (categories, project)
- Security boundary at the API layer
