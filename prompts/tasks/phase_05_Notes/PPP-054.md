## 2. Objective

> Implement Category CRUD endpoints: Create, List, Update, Delete — user-scoped personal categories.

Categories are a personal taxonomy — each user manages their own set. CreateCategory adds a new category name for the current user (enforcing uniqueness per user). GetCategories returns the user's categories, optionally with note counts. UpdateCategory renames a category. DeleteCategory removes it (and the NoteCategory join records cascade). All operations are user-scoped — users cannot see or modify other users' categories. This task teaches user-scoped resource management and the pattern of personal vs shared data.

## 4. Technical Guidance

**Conventions for this module:**
- Commands: `CreateCategoryCommand` (Name), `UpdateCategoryCommand` (Id, Name), `DeleteCategoryCommand` (Id)
- Query: `GetCategoriesQuery` — returns user's categories, optionally with `noteCount`
- Handler create: verify name not duplicated for user; create Category; save
- Handler update: verify ownership; verify new name not duplicated; update; save
- Handler delete: verify ownership; remove (NoteCategory cascade); save
- Endpoints: `POST /api/v1/categories`, `GET /api/v1/categories`, `PUT /api/v1/categories/{id}`, `DELETE /api/v1/categories/{id}`
- Response: CategoryDto with `Id, Name, NoteCount`
- Uniqueness constraint: unique index on `(UserId, Name)` or application-level check

**Concepts practiced in this task:**
- User-scoped resource management
- Uniqueness within a scope (per-user uniqueness)
- Cascade delete behavior
