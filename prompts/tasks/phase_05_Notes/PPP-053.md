## 2. Objective

> Implement GetNoteById, UpdateNote, and DeleteNote commands with ownership authorization.

These commands complete the Notes CRUD lifecycle. GetNoteById returns the full note content including categories and project info. UpdateNote allows modifying title, content (re-sanitized on save), project association, and categories. DeleteNote removes the note and its category associations (cascade). All operations are scoped to the note owner. This task completes the standard resource lifecycle for the Notes module.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetNoteByIdQuery` — load note with categories eager loaded; verify ownership → 404 if not owner
- Commands: `UpdateNoteCommand` (Id, Title, Content, ProjectId?, CategoryIds?), `DeleteNoteCommand` (Id)
- Update handler: load note, verify ownership, sanitize content if changed, update fields, sync categories (remove all, add selected), save
- Delete handler: load note, verify ownership, remove (EF Core cascade removes NoteCategory associations), save
- Endpoints: `GET /api/v1/notes/{id}`, `PUT /api/v1/notes/{id}`, `DELETE /api/v1/notes/{id}`

**Concepts practiced in this task:**
- Full resource lifecycle (CRUD)
- Many-to-many relationship synchronization (category updates)
- Cascade delete behavior
