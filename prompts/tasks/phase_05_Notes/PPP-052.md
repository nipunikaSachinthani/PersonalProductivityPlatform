## 2. Objective

> Implement GetNotes list query with full-text search, category filtering, and pagination via `GET /api/v1/notes`.

The GetNotes endpoint supports searching across note titles and content — initially using SQL `LIKE` with a stretch goal of SQL Server Full-Text Search (FTS) for production-grade performance. Category filtering narrows results to notes tagged with specific categories. Results are paginated and include category names and project info. Only the user's own notes are returned (ownership authorization). This task teaches search implementation strategies, the tradeoff between LIKE and FTS, and performance-conscious query design.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetNotesQuery` with `Page`, `PageSize`, `Search?`, `CategoryId?`, `ProjectId?`, `SortBy? (createdAt/updatedAt/title)`
- Handler: filter by `UserId == currentUserId`; if search provided, filter `Title.Contains(search) || Content.Contains(search)`; if categoryId, filter by join through NoteCategories; if projectId, filter by ProjectId FK
- Search approach: use `LIKE '%term%'` for Phase 05; document FTS setup as a follow-up (requires SQL Server FTS catalog)
- Pagination: standard `Skip/Take` with total count
- Response: `ApiResponse<List<NoteDto>>` with meta; NoteDto includes `Title`, `Preview (first 200 chars of text content)`, `Categories (name list)`, `ProjectName`, `UpdatedAt`

**Concepts practiced in this task:**
- Search implementation (LIKE vs FTS tradeoffs)
- Multi-criteria filtering
- Content preview generation (strip HTML, truncate)
- Query performance awareness
