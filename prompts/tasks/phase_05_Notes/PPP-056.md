## 2. Objective

> Build the Notes list page with search bar (debounced), category filter chips, and note preview cards.

The Notes page is the content hub. A search bar at the top with 300ms debounce triggers API calls as the user types. Category filter chips below the search bar allow narrowing by one or more categories. Note cards display title, content preview (first 200 chars, HTML stripped), categories as badges, project name, and last-updated timestamp. Clicking opens the note detail/edit view. A "New Note" button opens the TipTap editor form. This task teaches debounced search, filter chip patterns, content preview rendering, and list-page UX patterns.

## 4. Technical Guidance

**Conventions for this module:**
- Page: `NotesPage` in `pages/NotesPage.tsx` at route `/notes`
- Search bar: controlled input, `useDebounce` hook (300ms), value drives `search` query param in `useQuery`
- Category filters: fetch categories via `useQuery`, render as shadcn `Badge` chips, toggle on/off, selected IDs drive `categoryId` query param
- Note cards: shadcn `Card` with title (h3), stripped-text preview (2 lines, `line-clamp-2`), category badges, project link, relative timestamp
- Empty state: "No notes found" with illustration; search-empty: "No notes match your search"
- Loading: skeleton cards (3-4)
- Click card → navigate to `/notes/:id` (detail/edit view)
- "New Note" button → navigates to `/notes/new`
- TanStack Query: query key includes search, categoryId, page params

**Concepts practiced in this task:**
- Debounced search input
- Filter chip UI pattern
- HTML content preview (strip tags, truncate)
- Relative timestamps ("2 hours ago")
