## 2. Objective

> Write Phase 05 light tests: search query handler test and XSS sanitization unit test.

The search query test verifies that the GetNotes handler correctly filters by search term (LIKE matching on title and content), by category, and returns paginated results. The XSS sanitization test proves that the content sanitizer strips `<script>` tags, `onclick` handlers, and other dangerous constructs while preserving safe HTML (paragraphs, bold, links). These tests validate the two riskiest parts of the Notes module — search correctness and security.

## 4. Technical Guidance

**Conventions for this module:**
- Unit test 1: `ContentSanitizerTests` — test cases: `<script>alert('xss')</script>` → stripped; `<p onclick="evil()">text</p>` → `onclick` removed; `<strong>bold</strong>` → preserved; `<a href="https://safe.com">link</a>` → preserved; empty string → empty string
- Handler test: `GetNotesQueryHandlerTests` — seed notes with varied content; search for term present in title → returned; search for term in content only → returned; search for term not present → empty; filter by category → only notes with that category returned; pagination works with total count
- Use EF Core InMemory provider for handler tests

**Concepts practiced in this task:**
- Security testing (XSS vectors)
- Search query verification
- Handler testing with seeded data
