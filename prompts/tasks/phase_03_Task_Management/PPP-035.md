## 2. Objective

> Build the Task List UI with shadcn Table, status/priority filters, sort controls, pagination, and empty/loading/skeleton states.

The Task List is the primary user interface — users will spend most of their time here. It renders a shadcn `Table` with columns for title, status (color-coded badge), priority (color-coded badge), due date (with overdue highlighting), and actions (complete, edit, delete). Filter dropdowns for status and priority update the TanStack Query parameters. Sort toggles on the due date column. Pagination controls at the bottom. Empty states show helpful messaging. Loading shows skeleton rows. This task teaches data-heavy UI composition, TanStack Query parameter management, and all UI states (loading, empty, error, data).

## 4. Technical Guidance

**Conventions for this module:**
- Page: `TasksPage` in `pages/TasksPage.tsx`
- Components: `TaskList` in `features/tasks/TaskList.tsx`, `TaskFilters` for filter controls, `TaskPagination` for page controls
- Data fetching: `useQuery` with query key `['tasks', { page, pageSize, status, priority, sortBy, sortOrder }]`
- Status badges: green for Done, blue for InProgress, gray for Todo (shadcn `Badge` component)
- Priority badges: red for Critical, orange for High, yellow for Medium, gray for Low
- Overdue: if `DueDate < today && Status !== Done`, show red date text
- Empty state: "No tasks yet. Create your first task!" with a CTA button
- Loading: shadcn `Skeleton` rows matching table structure
- Error: inline error message with retry button
- Action buttons: checkmark (complete), pencil (edit), trash (delete) per row

**Concepts practiced in this task:**
- TanStack Query with dynamic parameters
- shadcn/ui Table composition
- All UI states (loading, empty, error, edge cases)
- Color-coded status visualization
- Pagination UI pattern
