## 2. Objective

> Implement GetTaskById query and UpdateTask command with ownership authorization.

GetTaskById returns a single task by ID with ownership check — if the task doesn't belong to the current user, it returns 404 (not 403, to prevent information leakage about other users' tasks). UpdateTask allows modifying title, description, priority, and due date; status is updated separately via Complete/Reopen endpoints (PPP-034). Authorization ensures only the task owner can update it. This task completes the basic CRUD pattern and reinforces resource-level authorization beyond simple role checks.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetTaskByIdQuery` with `Id` → returns `TaskDto` or `Result<TaskDto>.NotFound()`
- Command: `UpdateTaskCommand` with `Id`, `Title`, `Description`, `Priority`, `DueDate?`
- Validator: same field rules as CreateTask
- Handler: load task by ID, verify `CreatedByUserId == currentUserId` (return 404 if not), update fields, set `UpdatedAt = UtcNow`, save
- Endpoints: `GET /api/v1/tasks/{id}`, `PUT /api/v1/tasks/{id}`
- Response: 200 with `TaskDto`; 404 if not found or not owned

**Concepts practiced in this task:**
- Resource-level authorization (ownership check)
- Security through obscurity (404 instead of 403 for non-owned resources)
- Full RESTful resource lifecycle
