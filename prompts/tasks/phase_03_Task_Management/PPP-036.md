## 2. Objective

> Build the Create/Edit Task form using shadcn Dialog/Sheet, React Hook Form + Zod, and TanStack Query mutations.

Tasks are created and edited via a modal dialog (shadcn `Dialog` or `Sheet`). The form reuses the Zod validation schema from the backend concepts: title (required, 1-200 chars), description (optional, max 2000), priority (select dropdown with Low→Critical), and due date (date picker). On create, TanStack Query mutation invalidates the task list cache so the table refreshes automatically. On edit, the form is pre-populated with existing task data. Success shows a toast notification. This task teaches modal forms, cache invalidation patterns, and optimistic UI updates.

## 4. Technical Guidance

**Conventions for this module:**
- Component: `TaskForm` in `features/tasks/TaskForm.tsx` (shared for create and edit)
- Form fields: `title` (Input), `description` (Textarea), `priority` (Select), `dueDate` (Input type="date" or shadcn Calendar)
- Zod schema in `features/tasks/taskSchema.ts`: `z.object({ title: z.string().min(1).max(200), description: z.string().max(2000).optional(), priority: z.enum(['Low','Medium','High','Critical']), dueDate: z.string().optional() })`
- Create mutation: `useMutation` calling `tasksApi.create(data)`, `onSuccess` invalidates `['tasks']` query and closes dialog
- Edit mutation: `useMutation` calling `tasksApi.update(id, data)`, `onSuccess` invalidates `['tasks']` and `['task', id]`
- Dialog: shadcn `Dialog` with `DialogTrigger` (button or row action), `DialogContent` wrapping the form, `DialogTitle`
- Toast: shadcn `Sonner` toast on success ("Task created", "Task updated")
- Loading: submit button disabled + spinner during mutation

**Concepts practiced in this task:**
- Modal form pattern
- TanStack Query cache invalidation
- Shared form component (create + edit)
- Toast notification integration
- Zod schema reuse
