## 2. Objective

> Create the Tasks table schema with EF Core migration: Id, Title, Description, Status, Priority, DueDate, CreatedByUserId, timestamps.

This task defines the data model for the platform's most central entity — Tasks. The schema stores the task's core attributes (title, description), workflow state (Todo/InProgress/Done status), importance (Low/Medium/High/Critical priority), an optional due date, and ownership (CreatedByUserId FK to Users). Status and Priority are stored as enums in the Domain layer with string conversion in EF Core, teaching the enum-as-string persistence pattern and separation of domain concepts from storage details.

## 4. Technical Guidance

**Conventions for this module:**
- Entity: `TaskItem` in `Domain/Entities/` (named `TaskItem` to avoid collision with `System.Threading.Tasks.Task`)
- Fields: `Id (int)`, `Title (string, max 200)`, `Description (string, max 2000)`, `Status (TaskStatus enum)`, `Priority (TaskPriority enum)`, `DueDate (DateTime?, UTC)`, `CreatedByUserId (int, FK→Users)`, `CreatedAt (DateTime, UTC)`, `UpdatedAt (DateTime, UTC)`, `CompletedAt (DateTime?, UTC)`
- Enums in `Domain/Enums/`: `TaskStatus { Todo, InProgress, Done }`, `TaskPriority { Low, Medium, High, Critical }`
- EF config: store enums as strings via `HasConversion<string>()`; configure indexes on `CreatedByUserId`, `Status`, `DueDate`
- Table name: `Tasks`
- `CreatedByUserId` is non-nullable — every task has an owner

**Concepts practiced in this task:**
- Domain entity design
- Enum persistence strategies
- EF Core index configuration for query performance
- UTC timestamp convention
