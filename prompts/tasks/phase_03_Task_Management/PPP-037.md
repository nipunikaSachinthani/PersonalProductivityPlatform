## 2. Objective

> Build the Task Detail view with full task information display, complete/reopen actions, and inline editing.

The Task Detail page provides a focused view of a single task — title, description, priority badge, status badge, due date, and creation metadata. It supports inline actions: the complete button transitions the task to Done; reopen moves it back. An edit button opens the same form dialog from PPP-036 pre-populated. A delete button with confirmation dialog removes the task. Navigation back to the list is via breadcrumb or back button. This task teaches detail view patterns, optimistic updates for status changes, and confirmation dialogs for destructive actions.

## 4. Technical Guidance

**Conventions for this module:**
- Page: `TaskDetailPage` in `pages/TaskDetailPage.tsx` at route `/tasks/:id`
- Data fetching: `useQuery(['task', id], () => tasksApi.getById(id))`
- Mutations: `useMutation` for complete, reopen, delete — each invalidates task detail and task list caches
- UI: shadcn `Card` layout with sections for title (h1), metadata grid (status, priority, due date, created), description block
- Status actions: "Mark Complete" button (shown when status != Done), "Reopen" button (shown when status == Done)
- Delete: shadcn `AlertDialog` for confirmation, then delete mutation + navigate back to list
- Loading: skeleton card; error: "Task not found" message; empty: not applicable (404 handled by error state)
- Breadcrumb: Tasks > [Task Title] using React Router

**Concepts practiced in this task:**
- Detail view with action mutations
- Optimistic updates with cache invalidation
- Confirmation dialog pattern
- Route parameters in React Router
