## 2. Objective

> Implement CreateTask MediatR command with FluentValidation and `POST /api/v1/tasks` endpoint.

The CreateTask command is the first write operation in the task module. The handler maps the DTO to a TaskItem entity, sets the current user as owner (extracted from JWT claims), initializes status to `Todo`, sets UTC timestamps, and persists via EF Core. FluentValidation ensures title is required and within length limits, description doesn't exceed max length, priority is a valid enum value, and due date (if provided) is in the future. The controller returns 201 Created with the task DTO. This task teaches the full write-side CQRS pipeline in the context of a real domain entity.

## 4. Technical Guidance

**Conventions for this module:**
- Command: `CreateTaskCommand` in `Application/Tasks/Commands/` with `Title`, `Description`, `Priority`, `DueDate?`
- Validator: `CreateTaskCommandValidator` ‚Äî Title required, 1-200 chars; Description max 2000; Priority must be valid enum; DueDate if provided must be > UtcNow
- Handler: `CreateTaskCommandHandler` ‚Äî map command ‚Üí `TaskItem`, set `CreatedByUserId` from `ICurrentUserService`, set `CreatedAt`/`UpdatedAt` to UtcNow, set `Status = Todo`, save to DbContext, return `TaskDto`
- `ICurrentUserService` interface in Application layer, implementation in Api layer using `IHttpContextAccessor`
- Controller: `TasksController.Create()` ‚Üí `POST /api/v1/tasks` with `[Authorize]`
- Response: 201 Created with `ApiResponse<TaskDto>`, location header
- `TaskDto`: Id, Title, Description, Status, Priority, DueDate, CreatedAt, UpdatedAt (no sensitive data)

**Concepts practiced in this task:**
- CQRS command pipeline
- FluentValidation with domain rules
- Current user extraction from JWT
- Structured API responses (envelope pattern)
- UTC timestamp discipline
