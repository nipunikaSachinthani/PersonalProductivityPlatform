## 2. Objective

> Add test execution to CI pipeline and create CD workflow that deploys to Azure App Service on merge to main.

The CI pipeline is extended with test execution — `dotnet test` and `npm test` run after builds pass, with test results published as artifacts. A separate CD workflow triggers on merge to `main`: it downloads build artifacts, deploys the .NET API to Azure App Service, deploys the React app to Azure Static Web Apps (or App Service), and runs EF Core migrations against Azure SQL. GitHub Secrets store connection strings and JWT keys. This task teaches the complete CI/CD pipeline, environment-based deployment, and secrets management in CI.

## 4. Technical Guidance

**Conventions for this module:**
- CI extension: add `test-backend` job (depends on build-backend) — `dotnet test --logger trx`; add `test-frontend` job (depends on build-frontend) — `npm test`
- CD workflow: `.github/workflows/cd.yml` — trigger on `push` to `main`
- CD jobs: deploy API (`azure/webapps-deploy@v2` action), deploy React (`azure/static-web-apps-deploy@v1` or `azure/webapps-deploy@v2`), run migrations (Azure SQL connection)
- GitHub Secrets: `AZURE_WEBAPP_PUBLISH_PROFILE`, `AZURE_SQL_CONNECTION_STRING`, `JWT_KEY`, `BLOB_CONNECTION_STRING`
- Environments: `dev` environment in GitHub (add approval gates later for `staging`/`prod`)
- Health check: after deploy, curl the health endpoint to verify deployment

**Concepts practiced in this task:**
- Full CI/CD pipeline
- Environment-based deployment
- Secrets management in GitHub Actions
- Database migrations in deployment
