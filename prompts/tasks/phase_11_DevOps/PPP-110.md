## 2. Objective

> Create GitHub Actions CI workflow: build .NET solution and React app on every push and PR.

The CI pipeline is the first automated quality gate. On every push to any branch and every PR to main, GitHub Actions builds both the .NET backend (`dotnet build`) and the React frontend (`npm ci && npm run build`), caching dependencies for faster subsequent runs. A matrix build could target multiple .NET versions, but .NET 8 is sufficient. This task teaches CI/CD as code, GitHub Actions workflow syntax, dependency caching, and the practice of "build on every change."

## 4. Technical Guidance

**Conventions for this module:**
- Workflow file: `.github/workflows/ci.yml`
- Trigger: `push` to any branch, `pull_request` to `main`
- Jobs:
  - `build-backend`: checkout, setup .NET 8, restore, build, upload artifacts
  - `build-frontend`: checkout, setup Node 20, `npm ci`, `npm run build`, upload artifacts
- Caching: NuGet packages via `actions/cache`; npm via `actions/setup-node` built-in cache
- Artifacts: upload build outputs for use in deploy job (PPP-112)
- Run jobs in parallel (backend and frontend independent)

**Concepts practiced in this task:**
- CI/CD as code (GitHub Actions)
- Dependency caching for build speed
- Artifact publishing
- Parallel job execution
