## 2. Objective

> Build the Login page in React using shadcn/ui form components, React Hook Form + Zod validation, and TanStack Query mutation.

The Login page is the first real frontend screen users see. It uses shadcn/ui `Card`, `Input`, `Button`, and `Form` components styled with Tailwind. React Hook Form manages form state and submission; Zod validates email format and password presence before the request is sent. A TanStack Query `useMutation` calls the login API, stores the tokens (in memory + optionally localStorage for refresh token), and redirects to the dashboard on success. Validation errors from the API (401, 400) are displayed inline. This task teaches form architecture, API integration with TanStack Query mutations, and token management on the client.

## 4. Technical Guidance

**Conventions for this module:**
- Component: `LoginPage` in `pages/LoginPage.tsx`
- Form: `react-hook-form` with `zod` resolver, fields: email (required, valid email), password (required, min 8)
- API call: `authApi.login(email, password)` using axios, wrapped in `useMutation`
- On success: store access token in memory (React state/context), refresh token in localStorage; redirect to `/dashboard`
- On error: display error message from API response below the form
- UI: centered card layout on a gradient/minimalist background; "Productivity Platform" branding
- Loading state: button shows spinner and disables during mutation
- Link to Register page at bottom
- File location: `features/auth/LoginForm.tsx` (form component), `services/authApi.ts` (API client)

**Concepts practiced in this task:**
- React Hook Form + Zod validation
- TanStack Query mutations
- Client-side token management
- shadcn/ui form composition
