## 2. Objective

> Build the Registration page in React with shadcn/ui form, validation, and successful-redirect-to-login flow.

The Registration page collects first name, last name, email, and password (with confirmation field), validates on the client with Zod (password match, strength requirements), submits via TanStack Query mutation, and on success redirects the user to the login page with a "Registration successful" toast. This task completes the auth form set and reinforces the form patterns established in PPP-017, while adding password confirmation validation and inline strength indicators.

## 4. Technical Guidance

**Conventions for this module:**
- Component: `RegisterPage` in `pages/RegisterPage.tsx`
- Form fields: firstName, lastName, email, password, confirmPassword
- Zod schema: email valid format, password min 8 with at least 1 uppercase + 1 number + 1 special char, confirmPassword must match password
- API: `POST /api/v1/auth/register` via `useMutation`
- On success: show toast "Account created! Please log in." and navigate to `/login`
- On duplicate email (409): display "An account with this email already exists"
- UI: same centered card layout as login; link to Login page
- Error display: field-level errors from Zod, API-level errors below form

**Concepts practiced in this task:**
- Multi-field Zod validation (cross-field: password match)
- API error mapping to form fields
- Post-registration redirect pattern
- Toast notifications (shadcn Sonner/Toast)
