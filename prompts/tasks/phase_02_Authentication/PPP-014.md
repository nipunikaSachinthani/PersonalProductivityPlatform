## 2. Objective

> Implement forgot password and reset password flow: `ForgotPassword` and `ResetPassword` commands with email stub.

Password reset is a critical self-service flow. The forgot-password endpoint accepts an email and generates a time-limited reset token (stored hashed in the database). In production, this token is emailed; for this learning project, the token is returned in the API response (stub) or logged. The reset-password endpoint accepts the token and new password, validates the token hasn't expired, and updates the password hash. This task teaches secure token generation, time-limited tokens, and the two-step reset flow that prevents account enumeration.

## 4. Technical Guidance

**Conventions for this module:**
- Add `PasswordResetToken` and `PasswordResetTokenExpiry` to User entity (nullable)
- Commands: `ForgotPasswordCommand` (Email), `ResetPasswordCommand` (Email, Token, NewPassword)
- Forgot handler: find user by email, generate cryptographically random token (base64), set expiry (1 hour), return token in response (log it for dev; stub for email)
- Reset handler: find user by email, verify token matches and hasn't expired, hash new password, clear token fields
- Response: Forgot → 200 OK always (prevents email enumeration); Reset → 200 OK or 400 if token invalid/expired
- Endpoints: `POST /api/v1/auth/forgot-password`, `POST /api/v1/auth/reset-password`

**Concepts practiced in this task:**
- Secure token generation (cryptographic randomness)
- Time-limited tokens
- Account enumeration prevention
- Two-step password reset flow
