## 2. Objective

> Write the Phase 02 light tests: password hasher unit test, login handler test, register API integration test, and login form component test.

This task validates all auth work with focused tests targeting the most critical paths. A unit test confirms password hashing works (hash ‚âÝ plaintext, verification succeeds). A handler test verifies login returns a token for valid credentials and 401 for invalid ones (using mocked dependencies). An integration test with WebApplicationFactory goes through the full register endpoint ‚Äî validation, hashing, DB persistence, response. A Vitest + React Testing Library test confirms the login form renders, validates inputs, and calls the API on submit. These four tests form the safety net for auth regressions and establish testing patterns used in every phase.

## 4. Technical Guidance

**Conventions for this module:**
- Unit test 1: `PasswordHasherTests` ‚Äî `HashPassword` produces different string, `VerifyHashedPassword` succeeds for correct password, fails for wrong
- Handler test: `LoginUserCommandHandlerTests` ‚Äî mock `UserManager` or repository, verify handler returns token DTO; verify throws/returns error for invalid password
- Integration test: `AuthControllerTests` using `WebApplicationFactory` ‚Äî POST to register with valid data ‚Üí 201 with UserDto; POST with duplicate email ‚Üí 409
- Component test: `LoginForm.test.tsx` using Vitest + RTL ‚Äî renders email/password inputs, shows validation errors on empty submit, calls mock API on valid submit
- Test projects setup from PPP-002 should already exist; add test packages as needed (`Moq`, `Microsoft.AspNetCore.Mvc.Testing`, `vitest`, `@testing-library/react`)

**Concepts practiced in this task:**
- Unit testing with mocking
- Integration testing with WebApplicationFactory
- React component testing with React Testing Library
- Test pyramid implementation
