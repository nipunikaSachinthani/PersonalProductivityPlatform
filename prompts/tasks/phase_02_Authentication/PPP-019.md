## 2. Objective

> Build the Forgot Password page, AuthContext with token/refresh interceptor, protected route wrapper, and AppShell layout.

This task ties the auth frontend together. The Forgot Password page completes the auth form set. The AuthContext provides user state, login/logout/register functions, and the current user object to the entire app. An Axios interceptor attaches the JWT to every request and automatically refreshes on 401 — the user never notices token expiry. The ProtectedRoute wrapper redirects to `/login` if unauthenticated. The AppShell layout provides the sidebar/navbar structure that all future pages live within. This task teaches React context patterns, HTTP interceptors, route guards, and layout composition.

## 4. Technical Guidance

**Conventions for this module:**
- `AuthContext` in `features/auth/AuthContext.tsx` — provides `user`, `login`, `logout`, `register`, `isLoading`, `isAuthenticated`
- On app mount: check for stored refresh token, call `/users/me` to hydrate user state; if 401, clear tokens
- Axios interceptor in `services/apiClient.ts`:
  - Request: attach `Authorization: Bearer <accessToken>`
  - Response 401: attempt `/auth/refresh-token`, retry original request; if refresh fails, call logout
- `ProtectedRoute` in `routes/ProtectedRoute.tsx` — renders `<Outlet />` if authenticated, `<Navigate to="/login" />` if not
- `AppShell` in `layouts/AppShell.tsx` — sidebar nav (placeholder for now), header with user name + logout, main content area
- `ForgotPasswordPage` in `pages/ForgotPasswordPage.tsx` — email input only; on submit show "If an account exists, a reset link has been sent"
- React Router setup: public routes (`/login`, `/register`, `/forgot-password`), protected routes (wrapped in `ProtectedRoute`)

**Concepts practiced in this task:**
- React Context for auth state
- Axios interceptors for token management
- Route guards / protected routes
- Layout composition pattern
- Silent token refresh
