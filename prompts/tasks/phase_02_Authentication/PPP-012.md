## 2. Objective

> Implement user login: `LoginUser` MediatR command, JWT access token generation, and `POST /api/v1/auth/login` endpoint.

Login verifies credentials and issues a JWT access token (15-minute expiry) that the frontend attaches to all subsequent API requests. The handler validates the email exists, verifies the password hash, checks the account is active, generates a JWT with claims (sub, email, roles), and returns both the access token and a refresh token. JWT configuration (key, issuer, audience, expiry) is read from User Secrets/appsettings — never hardcoded. This task teaches JWT fundamentals, claims-based identity, and the critical security boundary between authentication and authorization.

## 4. Technical Guidance

**Conventions for this module:**
- Command: `LoginUserCommand` with `Email`, `Password`
- JWT configuration in `appsettings.json` (Jwt:Key, Jwt:Issuer, Jwt:Audience: "ProductivityPlatform")
- JWT Key stored in User Secrets for development; 256-bit minimum
- Access token expiry: 15 minutes; Refresh token expiry: 7 days
- JWT claims: `sub` (user id), `email`, `name`, `role` (array)
- Token generation via `JwtSecurityTokenHandler` + `SymmetricSecurityKey`
- Response: `{ accessToken, refreshToken, expiresAt, user: UserDto }`
- Handler returns 401 Unauthorized for invalid credentials or inactive account
- Add JWT authentication services in `Program.cs`: `AddAuthentication().AddJwtBearer()`

**Concepts practiced in this task:**
- JWT token generation and structure
- Claims-based identity
- Symmetric key security
- Token lifetime management
