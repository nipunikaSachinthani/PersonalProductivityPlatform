## 2. Objective

> Implement `GET /api/v1/users/me` endpoint returning the authenticated user's profile.

This endpoint is the frontend's primary way to know "who is logged in." After login, the React app calls `/users/me` with the JWT to get the current user's id, name, email, and roles. It's protected by `[Authorize]` â€” only a valid JWT can access it. The handler extracts the user ID from the JWT `sub` claim, queries the database, and returns a `UserDto`. This task is small but essential: it validates the entire auth pipeline (registration â†’ login â†’ token â†’ authorization â†’ data access) from a real consumer's perspective.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetCurrentUserQuery` extracting user ID from `IHttpContextAccessor` or claims principal
- Endpoint: `UsersController.GetMe()` â†’ `GET /api/v1/users/me` with `[Authorize]`
- Extract user ID: `User.FindFirst(ClaimTypes.NameIdentifier)?.Value` or `User.FindFirst("sub")?.Value`
- Return: `ApiResponse<UserDto>` with 200 OK; 401 if no token; 404 if user deleted
- Include roles in UserDto for frontend authorization decisions

**Concepts practiced in this task:**
- Authenticated user context extraction
- Claims-based identity in API consumers
- Profile endpoint pattern
