## 2. Objective

> Create the identity database schema: Users, Roles, UserRoles, RefreshTokens tables with EF Core migration.

This task establishes the authentication data layer â€” the tables that store user accounts, roles, role assignments, and refresh tokens. Every secured endpoint in subsequent phases depends on this schema. The Users table stores credentials (hashed passwords only), profile data, and account status. Roles and UserRoles enable role-based access control (RBAC). RefreshTokens stores short-lived tokens that allow silent session renewal without re-entering credentials. This task teaches relational database design for identity, EF Core entity configuration, and migration management.

## 4. Technical Guidance

**Conventions for this module:**
- Entities in `ProductivityPlatform.Domain/Entities/`: `User`, `Role`, `UserRole`, `RefreshToken`
- User fields: `Id (int)`, `FirstName`, `LastName`, `Email`, `PasswordHash`, `IsActive`, `CreatedAt`
- Role fields: `Id (int)`, `Name` (string)
- UserRole: composite key on `UserId` + `RoleId`
- RefreshToken: `Id (int)`, `UserId` (FK), `Token` (string), `ExpiryDate`, `CreatedAt`, `RevokedAt` (nullable)
- Seed default roles in migration: "Admin", "User"
- All timestamps use `DateTime.UtcNow`
- Table names: PascalCase plural (`Users`, `Roles`, `UserRoles`, `RefreshTokens`)

**Concepts practiced in this task:**
- Relational database design for identity
- EF Core entity configuration (Fluent API)
- Composite keys and foreign key relationships
- Data seeding in migrations
