## 2. Objective

> Create a health check endpoint (`GET /api/v1/health`) and a landing page in React that calls it, proving end-to-end connectivity.

This is the first "vertical slice" of the platform — a request from the browser through the React app, across the network, into the .NET API, hitting the database, and returning a response. It verifies that every piece of infrastructure from Phase 01 works together: the React dev server, CORS configuration, API routing, DI container, DbContext, and Serilog logging. The health endpoint returns `{ status: "Healthy", timestamp, database: "Connected" }` after verifying the database is reachable. The landing page displays "Platform Online" using TanStack Query to fetch from the health endpoint.

## 4. Technical Guidance

**Conventions for this module:**
- API endpoint: `GET /api/v1/health` in `HealthController` under `ProductivityPlatform.Api/Controllers/`
- Response format: `{ "status": "Healthy", "timestamp": "2026-07-03T...", "database": "Connected" }`
- Database health check: attempt a lightweight query (`SELECT 1` or `Database.CanConnectAsync()`)
- React: `useQuery` from TanStack Query in `LandingPage` component under `pages/`
- Display a green status badge when healthy, red when unreachable
- API base URL from environment variable: `VITE_API_BASE_URL=http://localhost:5000/api/v1`
- Use the `ApiResponse` envelope pattern: `{ data, errors, meta }`

**Concepts practiced in this task:**
- End-to-end vertical slice
- TanStack Query for API data fetching
- Health check pattern
- CORS debugging
