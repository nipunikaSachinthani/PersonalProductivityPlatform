## 2. Objective

> Initialize the Git repository with the full project folder structure, .gitignore, and editor configuration.

This is the first task of the entire project — it establishes the foundation every subsequent task builds upon. A clean, well-organized repository from day one prevents technical debt accumulation and sets professional standards. This task teaches modern Git hygiene (trunk-based development, `.gitignore` best practices, EditorConfig for cross-editor consistency) and ensures every team member or future contributor has a predictable project layout.

## 4. Technical Guidance

**Conventions for this module:**
- Trunk-based development: `main` branch + short-lived `feature/PPP-NNN-slug` branches
- `.gitignore` must cover: `node_modules/`, `bin/`, `obj/`, `.vs/`, `*.user`, `appsettings.*.local.json`, `.env*`, `dist/`, `coverage/`
- EditorConfig: 4-space indent for C#, 2-space for JS/TS/JSON, UTF-8, LF line endings
- Repository root must mirror the ARCHITECTURE_ANALYSIS.md folder structure (empty `backend/` and `frontend/` directories)

**Concepts practiced in this task:**
- Trunk-based development workflow
- Repository structure as architectural documentation
- Cross-editor consistency via EditorConfig
