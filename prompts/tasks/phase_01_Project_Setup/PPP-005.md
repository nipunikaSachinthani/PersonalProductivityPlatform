## 2. Objective

> Configure Swagger/OpenAPI, Serilog structured logging, and multi-environment appsettings in the .NET API project.

This task sets up the developer experience and observability foundations for the entire project. Swagger provides interactive API documentation visible at `/swagger` — essential for testing endpoints during development. Serilog with structured logging replaces the default console logger, enabling JSON-formatted logs in production (Phase 12) and human-readable console output during development. Multi-environment `appsettings` files (`Development`, `Production`) teach environment-aware configuration — a pattern that prevents hardcoded values and supports the CI/CD pipeline in Phase 11.

## 4. Technical Guidance

**Conventions for this module:**
- Swagger: `Swashbuckle.AspNetCore`, configure in `Program.cs` with `AddSwaggerGen()` and `UseSwagger()`/`UseSwaggerUI()`
- Swagger title: "Productivity Platform API", version "v1"
- Serilog: `Serilog.AspNetCore`, configure in `Program.cs` with `UseSerilog()`, write to Console in dev, JSON file in prod
- `appsettings.Development.json` for LocalDB connection, verbose logging
- `appsettings.Production.json` stub for later Azure configuration
- CORS policy: allow origin `http://localhost:5173` (Vite dev server) explicitly

**Concepts practiced in this task:**
- OpenAPI/Swagger for API documentation
- Structured logging with Serilog
- Environment-based configuration management
- CORS policy configuration
