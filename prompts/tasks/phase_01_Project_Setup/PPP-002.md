## 2. Objective

> Scaffold the .NET 8 Clean Architecture solution with four projects: Api, Application, Domain, and Infrastructure.

This task creates the backend skeleton following Clean Architecture — the structural pattern that governs all subsequent backend work. Each project has a clear dependency direction: Domain depends on nothing, Application depends on Domain, Infrastructure implements Domain/Application interfaces, and Api wires everything together. Getting this right now prevents circular dependencies and architectural drift later. This task teaches solution structure, project references, and the dependency inversion principle at the mechanical level.

## 4. Technical Guidance

**Conventions for this module:**
- Solution name: `ProductivityPlatform`
- Project names: `ProductivityPlatform.Api`, `ProductivityPlatform.Application`, `ProductivityPlatform.Domain`, `ProductivityPlatform.Infrastructure`
- Class library projects for Application and Domain; Web API project for Api; class library for Infrastructure
- Reference chain: Api → Application + Infrastructure; Application → Domain; Infrastructure → Application + Domain
- Target `net8.0` across all projects
- Add NuGet packages: `MediatR` (Application), `FluentValidation` (Application), `Serilog.AspNetCore` (Api), `Swashbuckle.AspNetCore` (Api)
- Tests projects: `ProductivityPlatform.UnitTests` and `ProductivityPlatform.IntegrationTests` under `backend/tests/`

**Concepts practiced in this task:**
- Clean Architecture project structure
- Dependency inversion via project references
- Solution-level organization
