## 2. Objective

> Configure EF Core with SQL Server LocalDB, create the initial DbContext, and generate the first empty migration.

This task wires the database layer into the Clean Architecture stack. Entity Framework Core is configured in the Infrastructure project with a connection string pointing to SQL Server LocalDB (the ideal zero-config dev database on Windows). The DbContext is registered in the DI container, and an initial migration proves the pipeline works end-to-end. This task teaches EF Core configuration, connection string management via User Secrets, and the code-first migration workflow — skills used in every subsequent data phase.

## 4. Technical Guidance

**Conventions for this module:**
- `AppDbContext` class in `ProductivityPlatform.Infrastructure/Persistence/`
- Connection string stored in User Secrets (`dotnet user-secrets set "ConnectionStrings:DefaultConnection" "..."`)
- LocalDB connection string: `Server=(localdb)\\MSSQLLocalDB;Database=ProductivityPlatform;Trusted_Connection=True;TrustServerCertificate=True`
- Register DbContext in `Program.cs` with `builder.Services.AddDbContext<AppDbContext>(...)`
- EF Core packages: `Microsoft.EntityFrameworkCore.SqlServer`, `Microsoft.EntityFrameworkCore.Tools`
- Initial migration: `dotnet ef migrations add InitialCreate` (creates empty database)
- Migration output folder: `Persistence/Migrations/`

**Concepts practiced in this task:**
- EF Core configuration and dependency injection
- Connection string management via User Secrets
- Code-first migrations workflow
- LocalDB as a zero-config dev database
