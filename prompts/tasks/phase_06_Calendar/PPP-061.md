## 2. Objective

> Implement GetCalendar endpoint returning tasks with due dates and standalone events within a date range via `GET /api/v1/calendar?start=&end=`.

The calendar endpoint accepts a start and end date (ISO 8601) and returns a unified list of calendar items — tasks with due dates in the range (as read-only events) and standalone CalendarEvents created by the user. Each item includes a type discriminator (`task` or `event`) for the frontend to render appropriately. Only the user's own tasks and events are returned. This task teaches date-range query construction, union-query patterns, and API design for calendar UIs.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetCalendarQuery` with `StartDate (DateTime, required)`, `EndDate (DateTime, required)`
- Handler: query Tasks where `DueDate >= start && DueDate <= end && UserId == currentUserId`; query CalendarEvents where `StartDate >= start && EndDate <= end && UserId == currentUserId`
- Map to unified `CalendarItemDto` with discriminator: `{ type: "task"|"event", id, title, startDate, endDate?, allDay, status?, priority?, projectId?, taskId? }`
- Sort by start date ascending
- Endpoint: `GET /api/v1/calendar?start=2026-07-01T00:00:00Z&end=2026-07-31T23:59:59Z`
- Response: `ApiResponse<List<CalendarItemDto>>`
- Edge case: start > end → 400 validation error

**Concepts practiced in this task:**
- Date-range querying with EF Core
- Union/combined data sources into unified DTOs
- ISO 8601 date parameter parsing
- Type discriminator pattern
