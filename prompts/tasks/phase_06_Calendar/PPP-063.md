## 2. Objective

> Integrate FullCalendar with month, week, and day views displaying task due dates and standalone events.

FullCalendar is a powerful, interactive calendar library. This task configures it with three view modes (month grid, week time-grid, day time-grid), fetches events from the `GET /api/v1/calendar` endpoint via a callback, and renders task due dates (color-coded by priority) and standalone events (color-coded by type). Clicking a task event navigates to the task detail. Clicking a date opens a quick-create event dialog. This task teaches third-party calendar integration, view configuration, data callback patterns, and interactive timeline UX.

## 4. Technical Guidance

**Conventions for this module:**
- Library: `@fullcalendar/react`, `@fullcalendar/daygrid`, `@fullcalendar/timegrid`, `@fullcalendar/interaction`
- Component: `CalendarView` in `features/calendar/CalendarView.tsx`
- Views: `dayGridMonth` (default), `timeGridWeek`, `timeGridDay` — all via FullCalendar header buttons
- Data fetching: use `events` prop as async function calling `calendarApi.getEvents(start, end)` (dates from FullCalendar's `fetchInfo`)
- Event rendering: tasks → priority-colored (Critical=red, High=orange, etc.); standalone events → blue; all-day events → different shade
- `eventClick`: navigate to task detail (`/tasks/:id`) or event detail
- `dateClick`: open quick-create event dialog (shadcn Dialog with start date pre-filled)
- `eventDrop`: drag-to-reschedule (see PPP-064)
- Styling: Tailwind wrapper; FullCalendar theme adapted via CSS custom properties

**Concepts practiced in this task:**
- Third-party calendar library integration
- Callback-based data fetching
- Interactive timeline UX
- Color-coded event rendering
