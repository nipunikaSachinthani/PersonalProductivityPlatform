## 2. Objective

> Implement drag-to-reschedule for task due dates on the calendar.

When a user drags a task event to a different date on the calendar, the task's `DueDate` is updated via the UpdateTask API. FullCalendar's `eventDrop` callback fires, extracts the new date, and calls a mutation to update the task. This provides a visual, intuitive way to reschedule work without navigating to the task form. This task teaches drag-and-drop integration, optimistic UI updates on the calendar, and API-driven state changes from interactive components.

## 4. Technical Guidance

**Conventions for this module:**
- FullCalendar config: `editable={true}` on task events (standalone events editable too)
- `eventDrop` callback: extract `event.extendedProps.type` — if "task", extract new date from `event.start`, call `tasksApi.update(taskId, { dueDate: newDate.toISOString() })` via `useMutation`
- Mutation: invalidate `['calendar']` and `['tasks']` queries on success
- `eventResize` callback: optionally support for standalone events (resize to change duration)
- Error handling: if update fails, show toast error and revert calendar event (via `revert()` in FullCalendar)
- Only allow dragging task events (not other users' events) — CSS class to disable if needed

**Concepts practiced in this task:**
- Drag-and-drop event handling
- Calendar-driven API updates
- Optimistic update with revert on error
- Interactive UI with server state sync
