## 2. Objective

> Implement CalendarEvent CRUD endpoints: Create, Update, Delete standalone events.

Standalone calendar events are full-fledged entities (not tied to tasks) that users can create for meetings, deadlines, reminders, or any time-blocked activity. Create validates that end date is after start date (unless AllDay). Update modifies event details. Delete removes the event. All operations are user-scoped. This task completes the calendar data layer before the visual UI is built.

## 4. Technical Guidance

**Conventions for this module:**
- Commands: `CreateCalendarEventCommand` (Title, StartDate, EndDate, AllDay, TaskId?, ProjectId?), `UpdateCalendarEventCommand` (Id, Title, StartDate, EndDate, AllDay), `DeleteCalendarEventCommand` (Id)
- Validator: start before end (unless AllDay); title required 1-200; if TaskId provided, verify task ownership
- Handler: create/update/delete with user ownership check
- Endpoints: `POST /api/v1/calendar/events`, `PUT /api/v1/calendar/events/{id}`, `DELETE /api/v1/calendar/events/{id}`
- Response: EventDto with Id, Title, StartDate (ISO), EndDate (ISO), AllDay, TaskId, ProjectId

**Concepts practiced in this task:**
- Date validation (start < end)
- CRUD for temporal entities
- All-day event handling
