## 2. Objective

> Write Phase 06 light tests: date-range query unit test and calendar API integration test.

The date-range query test verifies that GetCalendarQuery correctly includes tasks with due dates within the range, excludes tasks outside the range, includes standalone events in range, and excludes events outside the range. The integration test goes through the full calendar flow: create a task with a due date, create a standalone event, query the calendar for that range, and assert both appear in the unified response with correct type discriminators. These tests validate the union-query logic that the calendar UI depends on.

## 4. Technical Guidance

**Conventions for this module:**
- Unit test: `GetCalendarQueryHandlerTests` — seed tasks with due dates inside/outside range; seed events inside/outside range; assert query returns correct count; assert type discriminators are correct; assert sort order (by start date)
- Integration test: `CalendarControllerTests` — create task with due date, create event, GET calendar for date range, verify response contains both items, verify task item has type "task" and event item has type "event"

**Concepts practiced in this task:**
- Date-range test scenarios
- Union/merge query testing
- Type discriminator verification
