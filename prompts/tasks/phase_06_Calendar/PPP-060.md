## 2. Objective

> Create CalendarEvents table with EF Core migration for standalone events and task-due-date integration.

The CalendarEvents table stores standalone time-blocked events and provides the data layer for the calendar view. Events have start/end dates, optional all-day flag, and optional links to Tasks and Projects. Task due dates can be displayed on the calendar by querying Tasks alongside CalendarEvents. This task teaches date-range data modeling, dual-source querying (events + tasks), and the pattern of enriching one domain's data with another's.

## 4. Technical Guidance

**Conventions for this module:**
- Entity: `CalendarEvent` in `Domain/Entities/`
- Fields: `Id (int)`, `Title (string, max 200)`, `StartDate (DateTime, UTC)`, `EndDate (DateTime, UTC)`, `AllDay (bool)`, `UserId (int, FK→Users)`, `TaskId? (int, FK→Tasks)`, `ProjectId? (int, FK→Projects)`, `CreatedAt (UTC)`
- `StartDate` and `EndDate` stored as UTC; frontend converts to local time for display
- Index on `UserId` + `StartDate` + `EndDate` for date-range queries
- Table: `CalendarEvents`
- Tasks do not get a separate events table — their `DueDate` is queried alongside events

**Concepts practiced in this task:**
- Date-range data modeling
- UTC storage for temporal data
- Dual-source query design (events + tasks)
