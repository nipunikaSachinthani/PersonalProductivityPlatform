## 2. Objective

> Build the Project sidebar/list UI and project creation dialog in the AppShell layout.

The project sidebar provides persistent navigation — a list of the user's projects in the AppShell layout, with the ability to quickly switch between projects. Each item shows the project name and an archived indicator. A "New Project" button opens a creation dialog (same modal pattern as tasks). Selecting a project navigates to its detail page. The sidebar integrates with the task list view to optionally filter by project. This task teaches sidebar navigation patterns, context-switching between resources, and stateful layout composition.

## 4. Technical Guidance

**Conventions for this module:**
- Component: `ProjectSidebar` in `features/projects/ProjectSidebar.tsx` — integrated into `AppShell` layout
- Data: `useQuery(['projects'], () => projectsApi.getAll())` — fetch user's projects
- UI: vertical list of project names with active highlight (based on current route `projects/:id`), grayed out if archived
- "New Project" button opens `ProjectFormDialog` (shadcn Dialog with name/description form + Zod validation)
- Creation invalidates projects query cache, navigates to new project page
- Filter integration: selecting a project filters the task list view (via URL search param or context)
- Empty state: "No projects yet. Create your first project!"
- Loading: skeleton list items
- Location: sidebar in `layouts/AppShell.tsx`

**Concepts practiced in this task:**
- Sidebar navigation pattern
- Context-aware layout composition
- URL-driven state (active project from route)
- Cache invalidation across queries
