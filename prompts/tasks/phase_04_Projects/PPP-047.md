## 2. Objective

> Build the Project Detail page with task list, member management panel, and project actions (edit, archive).

The Project Detail page is the collaboration hub. A header shows the project name, description, and action buttons (edit, archive). Below, a tab layout or sections toggle between the project's task list and the member management panel. The task list reuses the TaskList component from Phase 03 but scoped to the project's tasks. The member panel lists members with their roles and provides add/remove controls (Owner only). A project-level task summary (counts by status) appears in the header. This task teaches tabbed layouts, cross-component reuse, and role-aware UI rendering.

## 4. Technical Guidance

**Conventions for this module:**
- Page: `ProjectDetailPage` in `pages/ProjectDetailPage.tsx` at route `/projects/:id`
- Header: project name (h1), description, archive status badge, action buttons (Edit opens update dialog, Archive with confirmation)
- Task counts: "3 Todo · 2 In Progress · 1 Done" using API response data
- Tabs: shadcn `Tabs` with "Tasks" and "Members" tabs
- Tasks tab: reuse `TaskList` component but pass `projectId` prop to fetch `GET /api/v1/projects/{id}/tasks`
- Members tab: `MemberList` — table with name, email, role badge; add member button (Owner only); remove button per row (Owner only, hidden for self)
- Add member dialog: email input + role select (Member/Viewer)
- Edit dialog: reuse `ProjectFormDialog` from PPP-046
- Loading: skeleton for header and tabs; error: "Project not found"
- Role-aware UI: Owner sees edit/archive/add-remove buttons; Member sees content but not management; Viewer sees read-only

**Concepts practiced in this task:**
- Tabbed content layout
- Component reuse across contexts (TaskList)
- Role-aware conditional rendering
- Multi-panel page composition
