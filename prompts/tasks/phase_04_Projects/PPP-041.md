## 2. Objective

> Implement CreateProject MediatR command with validation and `POST /api/v1/projects` endpoint.

Creating a project automatically adds the creator as a member with the Owner role. The handler creates the Project entity, adds a ProjectMember record (UserId = creator, Role = Owner), and persists both in a single transaction via EF Core's unit of work. FluentValidation ensures the project name is required and unique per user. The controller returns 201 with the project DTO including the initial member. This task teaches transactional writes across related entities and domain invariant enforcement (every project must have exactly one Owner at creation).

## 4. Technical Guidance

**Conventions for this module:**
- Command: `CreateProjectCommand` with `Name`, `Description?`
- Validator: name required, 1-200 chars
- Handler: create `Project` entity with `OwnerUserId = currentUserId`, `CreatedAt = UtcNow`; create `ProjectMember` with `UserId = currentUserId`, `Role = Owner`; add both to DbContext; save
- Response: `ProjectDto` with members array containing the owner
- Endpoint: `POST /api/v1/projects` with `[Authorize]`
- ProjectDto includes: Id, Name, Description, OwnerUserId, IsArchived, CreatedAt, Members (list of MemberDto with UserId, UserName, Role)

**Concepts practiced in this task:**
- Transactional writes across related entities
- Domain invariant: project must have an owner
- Automatic side effects (add creator as member)
