## 2. Objective

> Build the file upload zone (drag-and-drop) and attachment list in the task detail view.

A drag-and-drop upload zone on the task detail page allows users to drop files directly or click to browse. Files show upload progress, and on success appear in an attachment list below. The list shows file name, type icon, size (formatted), and upload date, with download and delete action buttons. Authorization controls: only task owner/project member sees upload zone and delete buttons. This task teaches drag-and-drop UX, file preview/icon mapping, and upload progress UI.

## 4. Technical Guidance

**Conventions for this module:**
- Components: `FileUploadZone` (drag-drop area), `AttachmentList` (files for a task) in `features/attachments/`
- Upload zone: HTML5 drag-and-drop with `onDrop`/`onDragOver` handlers; styled border-dashed area; also supports click-to-browse via hidden `<input type="file">`
- Upload: `useMutation` calling `attachmentsApi.upload(taskId, file)`; show progress bar (Axios `onUploadProgress` or fetch with `ReadableStream`)
- Validation: file type check (show error for disallowed types), size check (max 10MB)
- Attachment list: table or card list with file icon (based on MIME type — PDF icon, image preview thumbnail, generic document icon), file name, formatted size (KB/MB), relative upload time
- Actions: download button (opens SAS URL in new tab), delete button with confirmation dialog
- Image preview: for image types, show thumbnail (50x50) using the download URL

**Concepts practiced in this task:**
- Drag-and-drop file upload
- Upload progress indication
- File type icon mapping
- Formatted file size display
