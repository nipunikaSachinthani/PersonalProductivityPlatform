## 2. Objective

> Implement file download (redirect to blob URL or stream) and delete attachment endpoints with authorization.

Download provides access to the file — either by redirecting to the blob URL (with a SAS token for security) or by streaming through the API. Delete removes the attachment record from the database and the blob from storage in a single operation. Authorization ensures only the task owner or project member can download or delete attachments. This task completes the attachment lifecycle and teaches secure file access patterns and coordinated deletion across multiple storage systems.

## 4. Technical Guidance

**Conventions for this module:**
- Commands: `GetAttachmentDownloadUrlQuery` (AttachmentId), `DeleteAttachmentCommand` (AttachmentId)
- Download: load attachment record, verify user owns task or is member; generate SAS token (5-minute expiry, read-only) and return URL; or stream through API via `BlobClient.DownloadStreamingAsync()` and return `FileStreamResult`
- SAS approach is lighter on API server (client downloads directly from blob); streaming is more secure (blob never exposed)
- Choose SAS token approach for this phase (teaches SAS concept)
- Delete: load attachment, verify authorization, delete blob from storage, remove record from DB; wrap in transaction (DB first, then blob — if blob fails, rollback DB)
- Endpoints: `GET /api/v1/attachments/{id}/download`, `DELETE /api/v1/attachments/{id}`
- List attachments: include in task detail response or separate `GET /api/v1/tasks/{taskId}/attachments`

**Concepts practiced in this task:**
- SAS token generation for secure file access
- Coordinated deletion (DB + blob)
- Transactional operations across storage systems
