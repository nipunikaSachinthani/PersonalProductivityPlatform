## 2. Objective

> Write Phase 09 light tests: file size validation (rejected >10MB) and unauthorized upload rejected test.

The file size test verifies that the upload validator rejects files exceeding the 10MB limit with a clear validation error. The unauthorized upload test creates a task owned by User A, then attempts to upload a file as User B (non-owner, non-member) and asserts 403 or 404. These tests validate the two primary security boundaries for file uploads: size limits and authorization.

## 4. Technical Guidance

**Conventions for this module:**
- Unit test: `FileSizeValidatorTests` — create a mock `IFormFile` with length > 10MB, assert validation error; mock file with valid size, assert no error
- Integration test: `AttachmentAuthorizationTests` — create task as User A; attempt upload as User B → 404 (ownership-based, not revealing task existence); attempt download/delete as User B → 404
- Mock `IBlobStorageService` for unit tests to avoid actual blob calls
- For integration tests with Azurite, ensure Azurite is running or mock the blob service

**Concepts practiced in this task:**
- File validation testing
- Authorization boundary testing for file operations
- Mocking external services in tests
