## 2. Objective

> Implement file upload endpoint with validation (file type, size limits), upload to Azure Blob (Azurite), and create Attachment record.

The upload endpoint accepts multipart form data, validates the file (allowed types: images, PDFs, documents; max size: 10MB), streams the file to Azure Blob Storage, generates the blob URL, and creates an Attachment record in the database linking the file to the task. Authorization ensures only the task owner or project member can upload. The handler uses streaming to avoid loading large files into memory. This task teaches multipart file handling in ASP.NET Core, streaming uploads to cloud storage, and security validation for user-uploaded files.

## 4. Technical Guidance

**Conventions for this module:**
- Command: `UploadAttachmentCommand` with `IFormFile File`, `TaskId`
- Validator: file not empty; content type in allowed list (`image/jpeg`, `image/png`, `image/gif`, `application/pdf`, `text/plain`, `application/msword`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`); size <= 10MB (10 * 1024 * 1024 bytes)
- Handler: verify user owns task or is project member; generate unique blob name (`{taskId}/{guid}-{fileName}`); upload stream to blob container; create Attachment record with blob URL; save both
- Blob container name: `task-attachments` (create if not exists on startup)
- Upload: `BlobClient.UploadAsync(stream, overwrite: false)` for streaming
- Endpoint: `POST /api/v1/tasks/{taskId}/attachments` (multipart form, file field name "file")
- Response: 201 with `AttachmentDto { Id, FileName, BlobUrl, ContentType, SizeBytes, UploadedAt }`
- Rate limit uploads: enforce a maximum of 5 attachments per task
- File name sanitization: remove special characters, limit length

**Concepts practiced in this task:**
- Multipart file handling
- Streaming uploads to cloud storage
- File type and size validation (security)
- Blob URL generation
