## 2. Objective

> Enhance Serilog for structured JSON logging in production, add custom metrics (login failures, task creation rate), and implement enhanced health checks (DB + blob).

Serilog is reconfigured to output JSON in production (machine-parseable for log aggregation) while keeping human-readable console output in development. Custom Application Insights metrics track business-level events: login failures (by user/IP), task creation rate, registration count. Enhanced health checks verify database connectivity (`DbContext.Database.CanConnectAsync()`) and blob storage accessibility. These observability improvements move the system from basic logging to production-ready monitoring — L3 maturity target for observability.

## 4. Technical Guidance

**Conventions for this module:**
- Serilog JSON: configure `WriteTo.Console(new CompactJsonFormatter())` when `ASPNETCORE_ENVIRONMENT == Production`
- Serilog enrichment: add `.Enrich.WithCorrelationId()`, `.Enrich.WithMachineName()`
- Custom metrics: inject `TelemetryClient`, call `telemetryClient.TrackEvent("LoginFailed", properties)` in login handler; `TrackMetric("TasksCreated", count)` periodically or on create
- Health checks: `Microsoft.Extensions.Diagnostics.HealthChecks` — add `AddDbContextCheck<AppDbContext>()` for DB, custom `BlobStorageHealthCheck` (attempt to list containers or check existence)
- Health endpoint response: `{ status, checks: { database: "Healthy", blobStorage: "Healthy" } }`
- Update `GET /api/v1/health` or add separate `/healthz` for Kubernetes-style probes

**Concepts practiced in this task:**
- Structured logging for production
- Custom business metrics
- Enhanced health checks
- Kubernetes-style readiness/liveness probes
