## 2. Objective

> Configure Application Insights for request tracking, dependency tracing, and exception monitoring in the .NET API.

Application Insights provides production-grade observability: every HTTP request is tracked with duration, status code, and correlation ID; every database/HTTP dependency call is traced; unhandled exceptions are captured with stack traces. The Application Insights SDK is configured in `Program.cs` with the instrumentation key from appsettings/User Secrets. The Azure portal dashboard shows live metrics, failed requests, and performance bottlenecks. This task teaches cloud-native observability, distributed tracing concepts, and production monitoring setup.

## 4. Technical Guidance

**Conventions for this module:**
- NuGet: `Microsoft.ApplicationInsights.AspNetCore`
- Configure in `Program.cs`: `builder.Services.AddApplicationInsightsTelemetry()`
- Instrumentation key from `appsettings.json` → `ApplicationInsights:ConnectionString`
- Auto-collects: requests, dependencies (SQL/HTTP), exceptions, traces
- Custom telemetry: add `TelemetryClient` for custom events/metrics (login failures, task creation rate — see PPP-121)
- Correlation: request correlation ID propagated through middleware
- Dashboard: Azure Portal → Application Insights → Live Metrics, Failures, Performance
- For local dev: Application Insights still works (data sent to Azure); or use `DeveloperMode` for reduced sampling

**Concepts practiced in this task:**
- Application Performance Monitoring (APM)
- Distributed tracing
- Telemetry auto-collection
- Cloud-native observability
