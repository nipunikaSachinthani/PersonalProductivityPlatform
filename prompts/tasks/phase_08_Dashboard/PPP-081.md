## 2. Objective

> Implement dashboard chart data endpoint: `GET /api/v1/dashboard/charts?range=30d` returning daily completion counts for the productivity trend chart.

While the summary endpoint gives a point-in-time snapshot, the charts endpoint provides trend data — daily task completions over a configurable date range (7d, 30d, 90d). The handler groups completed tasks by date and counts them, filling in zero-count days for continuous chart data. An optional per-project filter narrows the data. This endpoint powers the line chart in the dashboard UI and teaches time-series querying with date bucketing and gap-filling for charting.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetDashboardChartsQuery` with `RangeDays (int, default 30, max 365)`, `ProjectId?`
- Handler: query tasks where `CompletedAt >= startDate && (ProjectId == filter || filter is null)`, group by `CompletedAt.Value.Date`, count
- Generate date range from `startDate` to `today`; fill zeroes for days with no completions (ensures continuous chart data)
- Optionally, also return tasks created per day for a "created vs completed" comparison
- Response: `DashboardChartsDto { completions: [{ date, count }] }`
- Endpoint: `GET /api/v1/dashboard/charts?range=30d&projectId=1`
- Add `IMemoryCache` with 5-minute TTL for the chart data (avoids repeated heavy queries)

**Concepts practiced in this task:**
- Time-series date bucketing
- Gap-filling for chart data
- Caching with IMemoryCache
- Date-range parameter handling
