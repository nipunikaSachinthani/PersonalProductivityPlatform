## 2. Objective

> Write Phase 08 light test: dashboard summary query returns correct counts with seeded data.

This test seeds the database with a known set of tasks — some completed this week, some completed last week, some overdue, varying statuses and priorities — and asserts that the dashboard summary query returns exactly the expected counts for each metric. It validates the aggregation logic, the date-range boundaries (what counts as "this week"), and the group-by operations. This is a critical test because dashboard metrics are the most visible and relied-upon data in the app.

## 4. Technical Guidance

**Conventions for this module:**
- Integration test: `DashboardControllerTests` using `CustomWebApplicationFactory`
- Seed data: create tasks with specific attributes:
  - 3 completed this week (Status=Done, CompletedAt within current week)
  - 1 completed last week (should NOT appear in "this week" count)
  - 2 overdue (DueDate < now, Status != Done)
  - Mix of statuses (2 Todo, 1 InProgress, 4 Done)
  - Mix of priorities (1 Critical, 2 High, 1 Medium, 1 Low)
  - 2 tasks with a project, 3 without
- Assert summary response: `completedThisWeek = 3`, `overdue = 2`, `byStatus.Done = 4`, `byPriority.Critical = 1`
- Also test completion rate calculation: `doneCount / totalCount * 100`

**Concepts practiced in this task:**
- Seed-data-driven integration testing
- Aggregation assertion patterns
- Date-boundary testing
