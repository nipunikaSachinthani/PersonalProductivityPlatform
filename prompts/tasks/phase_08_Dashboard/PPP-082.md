## 2. Objective

> Build the Dashboard page with stat cards (completed this week, overdue, completion rate) and task breakdown by status/priority.

The Dashboard page is the landing page after login — a visual summary of productivity. Four stat cards across the top show: completed this week (green), overdue count (red, with warning icon), completion rate (percentage with progress bar), and total tasks. Below, two card sections show task breakdown by status (green/blue/gray bars) and by priority (color-coded counts). Numbers animate on load for a polished feel. The page fetches data from the summary endpoint using TanStack Query. This task teaches dashboard layout, stat card composition, and data-driven UI patterns.

## 4. Technical Guidance

**Conventions for this module:**
- Page: `DashboardPage` in `pages/DashboardPage.tsx` at route `/dashboard` (root redirect)
- Components: `StatCard` for individual metrics, `StatusBreakdown` / `PriorityBreakdown` for distribution
- Stat cards: shadcn `Card` with icon, label, value (large number), and subtitle (e.g., "+12% from last week")
- Status breakdown: horizontal bar or stacked list with colored indicators and counts
- Priority breakdown: colored badges with counts (red=Critical, orange=High, yellow=Medium, gray=Low)
- Completion rate: circular progress indicator or a linear progress bar (shadcn `Progress`)
- Loading: skeleton cards (4); error: retry button
- Data: `useQuery(['dashboard', 'summary'], ...)` — single query for all data

**Concepts practiced in this task:**
- Dashboard layout with stat cards
- Data visualization with simple CSS components
- Progress indicators
- Loading skeleton patterns for data-heavy pages
