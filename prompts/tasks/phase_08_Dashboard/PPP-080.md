## 2. Objective

> Implement Dashboard summary aggregation queries: `GET /api/v1/dashboard/summary` returning tasks completed this week, overdue count, and breakdowns by status and priority.

The dashboard summary endpoint aggregates task data across the user's workspace into actionable metrics. It returns: tasks completed this week (count), overdue tasks (due date < now, not done), tasks by status (Todo/InProgress/Done counts), tasks by priority (Low/Medium/High/Critical counts), and overall completion rate. Queries use EF Core's `GroupBy` and `Count` with appropriate `WHERE` clauses for the user's tasks only. Per-project breakdowns are returned as a nested array. This task teaches aggregation query design, GroupBy patterns in EF Core, and performance-conscious read models.

## 4. Technical Guidance

**Conventions for this module:**
- Query: `GetDashboardSummaryQuery` — no parameters (user from context)
- Handler: execute multiple aggregation queries:
  - `completedThisWeek`: tasks where `Status == Done && CompletedAt >= startOfWeek`
  - `overdue`: tasks where `DueDate < UtcNow && Status != Done`
  - `byStatus`: `GroupBy(t => t.Status).Select(g => new { Status, Count })`
  - `byPriority`: `GroupBy(t => t.Priority).Select(g => new { Priority, Count })`
  - `completionRate`: `doneCount * 100.0 / totalCount`
  - `byProject`: tasks with `ProjectId != null` grouped by project, with per-status counts
- Consider using raw SQL or EF Core queries; use `.AsNoTracking()` for read-only performance
- Response: `DashboardSummaryDto` with all metrics
- Endpoint: `GET /api/v1/dashboard/summary` with `[Authorize]`
- Add indexes on `CompletedAt` and `DueDate` if not already present (from PPP-030)

**Concepts practiced in this task:**
- Aggregation query design
- EF Core GroupBy and Count
- Read-optimized DTOs
- Performance indexing awareness
