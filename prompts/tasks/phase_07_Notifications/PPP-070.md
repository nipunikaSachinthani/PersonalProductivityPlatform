## 2. Objective

> Create Notifications table with EF Core migration and define domain event base classes.

The Notifications table stores in-app alerts generated by system events — task assignments, completions, and due-date reminders. The schema includes polymorphic fields (`RelatedEntityType`, `RelatedEntityId`) enabling notifications to link back to any entity type. Domain event base classes (`TaskAssignedEvent`, `TaskCompletedEvent`, `DueDateApproachingEvent`) are defined in the Domain layer, each carrying the relevant entity data needed by notification handlers. This task teaches event-driven design foundations, polymorphic entity references, and domain event structures.

## 4. Technical Guidance

**Conventions for this module:**
- Entity: `Notification` in `Domain/Entities/`
- Fields: `Id (int)`, `UserId (int, FK→Users)`, `Type (string — "TaskAssigned"/"TaskCompleted"/"DueDateReminder")`, `Title (string, max 200)`, `Message (string, max 1000)`, `IsRead (bool, default false)`, `CreatedAt (UTC)`, `RelatedEntityType (string — "Task"/"Project")`, `RelatedEntityId (int?)`
- Domain events in `Domain/Events/`: `TaskAssignedEvent` (TaskId, AssignedToUserId, AssignedByUserId, TaskTitle), `TaskCompletedEvent` (TaskId, CompletedByUserId, TaskTitle), `DueDateApproachingEvent` (TaskId, UserId, TaskTitle, DueDate)
- Events implement `INotification` from MediatR
- Table: `Notifications`; index on `UserId, IsRead` for efficient unread count queries

**Concepts practiced in this task:**
- Event-driven design (domain events)
- Polymorphic entity references
- Notification data modeling
