## 2. Objective

> Implement Hangfire background job that scans for tasks with due dates within 24 hours and publishes DueDateApproachingEvent for each.

Hangfire provides a job scheduler and dashboard within the application. A recurring job (runs hourly) queries tasks where `DueDate` is within the next 24 hours, `Status != Done`, and no notification has been sent recently (idempotency). For each matching task, it publishes a `DueDateApproachingEvent` which the notification handler (PPP-072) consumes to create the reminder notification. Hangfire stores job data in SQL Server (same database). This task teaches background job scheduling, recurring job patterns, cron expressions, and the Hangfire dashboard integration.

## 4. Technical Guidance

**Conventions for this module:**
- NuGet: `Hangfire`, `Hangfire.SqlServer` (or `Hangfire.Core` + storage)
- Configure Hangfire in `Program.cs`: `AddHangfire(config => config.UseSqlServerStorage(connectionString))`, `AddHangfireServer()`
- Dashboard: `app.UseHangfireDashboard()` at `/hangfire` (consider protecting with `[Authorize(Roles = "Admin")]`)
- Recurring job: `RecurringJob.AddOrUpdate<DueDateReminderJob>("due-date-reminder", job => job.ExecuteAsync(), Cron.Hourly)`
- Job class: `DueDateReminderJob` — inject `AppDbContext` + `IMediator`; query tasks due within 24h, not done, no notification in last 24h; publish `DueDateApproachingEvent` for each
- Idempotency: check Notifications table for existing reminder for this task in last 24h
- Ensure Hangfire tables are created (automatic or via migration)

**Concepts practiced in this task:**
- Background job scheduling with Hangfire
- Recurring job patterns
- Cron expressions
- Job dashboard for monitoring
