## 2. Objective

> Write Phase 07 light tests: event handler creates notification correctly and background job integration test (or unit test with mocked clock).

The event handler test verifies that `TaskCompletedNotificationHandler` creates a Notification record with correct Type, UserId, Title, and Message when receiving a `TaskCompletedEvent`. The background job test (or unit test with `TimeProvider`/mocked `DateTime`) verifies that `DueDateReminderJob` queries tasks due within 24 hours, publishes events for each, and skips tasks already notified. These tests validate the event-driven pipeline end-to-end: publish → handle → persist.

## 4. Technical Guidance

**Conventions for this module:**
- Unit test: `TaskCompletedNotificationHandlerTests` — create handler with in-memory DbContext, send `TaskCompletedEvent`, assert notification created with Type="TaskCompleted", correct UserId, message contains task title
- Job test: `DueDateReminderJobTests` — seed tasks with due dates (today, tomorrow, next week, already done, already notified); run job; assert only tasks due within 24h and not done/notified get events published; use mocked `IMediator` to verify publish calls
- For the job test, use `TimeProvider` (System) or abstract `IDateTime` interface to control "now"

**Concepts practiced in this task:**
- Event handler verification
- Background job testing
- Time abstraction for testability
