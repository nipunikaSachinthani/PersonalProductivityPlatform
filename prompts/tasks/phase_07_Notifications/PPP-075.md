## 2. Objective

> Build the notification bell component in the AppShell header with unread count badge and dropdown notification list.

A notification bell icon in the top navigation bar shows the unread count as a badge. Clicking opens a dropdown panel listing recent notifications (latest 20, unread highlighted). Each notification shows the type icon, title, message, relative timestamp, and a link to the related entity. Clicking a notification marks it as read and navigates to the entity. A "Mark all as read" button at the top clears all unread. The unread count is fetched periodically (polling every 60 seconds via TanStack Query `refetchInterval`). This task teaches real-time-ish UI patterns, dropdown menus, badge indicators, and polling strategies.

## 4. Technical Guidance

**Conventions for this module:**
- Component: `NotificationBell` in `features/notifications/NotificationBell.tsx`, placed in AppShell header
- Unread count: `useQuery(['notifications', 'unread-count'], ...)` or derive from notifications list if small
- Dropdown: shadcn `DropdownMenu` with `DropdownMenuTrigger` (bell icon + badge) and `DropdownMenuContent`
- Badge: shadcn `Badge` variant="destructive" with unread count; hidden when 0
- Dropdown content: list of `NotificationItem` components showing icon (based on type), title, message (truncated), "X minutes ago" timestamp
- Click notification: mark as read mutation, navigate to related entity URL
- "Mark all read" button at top of dropdown
- Empty dropdown: "No notifications" message
- Polling: `refetchInterval: 60000` on unread count query

**Concepts practiced in this task:**
- Badge indicator pattern
- Dropdown menu with dynamic content
- Polling for near-real-time updates
- Event-driven UI (notification → navigation)
