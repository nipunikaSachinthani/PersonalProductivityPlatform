## 2. Objective

> Document how to run tests (`dotnet test`, `npm test`), configure coverage thresholds (aim 70%+ on Application layer), and produce a test report.

The final task of Phase 10 consolidates the testing effort into documentation and CI-ready configuration. Write clear instructions in the project README on how to run backend tests, frontend tests, and all tests. Configure `coverlet` for .NET code coverage with a 70% threshold on the Application layer. Configure Vitest coverage with `c8` or `istanbul`. Generate and commit a sample coverage report showing current state. This task teaches test documentation, coverage tooling configuration, and the discipline of measurable quality gates.

## 4. Technical Guidance

**Conventions for this module:**
- Backend coverage: `coverlet.collector` NuGet package; add to test projects
- Run: `dotnet test /p:CollectCoverage=true /p:CoverletOutputFormat=cobertura /p:Threshold=70 /p:ThresholdType=line /p:Include="[ProductivityPlatform.Application]*"`
- Frontend coverage: Vitest coverage config in `vitest.config.ts` — `coverage.provider: 'v8'`, `coverage.reporter: ['text', 'lcov']`, thresholds for statements/branches
- README updates: add "Running Tests" section with commands for backend, frontend, and all
- Add npm script: `"test:coverage": "vitest run --coverage"`
- Document test pyramid structure: what goes in UnitTests vs IntegrationTests vs frontend component tests

**Concepts practiced in this task:**
- Code coverage configuration
- Coverage threshold as quality gate
- Test documentation for developer onboarding
