## 2. Objective

> Implement frontend component tests with Vitest + React Testing Library: forms, hooks, key pages. Configure MSW for API mocking.

This task brings frontend test coverage from zero to comprehensive. Component tests validate that forms (login, register, task create/edit, project create, note form) render correctly, show validation errors, and submit with correct data. Hook tests cover custom hooks like `useAuth`, `useDebounce`, `useTasks`. Page tests verify key pages (dashboard, task list, project detail) render with mocked data. MSW (Mock Service Worker) intercepts API calls at the network level, enabling tests that exercise the full component tree without a running backend.

## 4. Technical Guidance

**Conventions for this module:**
- Test framework: Vitest (`vitest`), React Testing Library (`@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`)
- MSW: `msw` for API mocking — define handlers in `src/mocks/handlers.ts`, server in `src/mocks/server.ts`
- Test setup: `src/test/setup.ts` — configure MSW server, mock ResizeObserver, mock IntersectionObserver
- Form tests per module: render form, find inputs by label/placeholder, type invalid data → assert error messages, type valid data + submit → assert mutation was called with correct payload
- Hook tests: `renderHook` from RTL to test custom hooks in isolation
- Page tests: render page wrapped in providers (QueryClient, Router, Auth), mock API responses, assert key elements rendered
- Snapshot tests: optional, not required

**Concepts practiced in this task:**
- Component testing with React Testing Library
- MSW for API mocking
- Form interaction testing
- Hook testing with renderHook
