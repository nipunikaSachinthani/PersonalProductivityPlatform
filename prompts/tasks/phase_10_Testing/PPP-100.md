## 2. Objective

> Backfill backend unit tests: all FluentValidation validators across all modules.

Phase 10 is the test coverage ramp-up. This task systematically backfills unit tests for every validator in the system — auth (register, login, forgot/reset password), tasks (create, update, complete, reopen), projects (create, update, members), notes (create, update, search), calendar (event create, date range query params), and dashboard (range validation). Each validator test covers: valid input passes, each validation rule has a failing case, edge cases (max lengths, boundary dates, null handling). Aim for >90% coverage on the Application layer's validation logic.

## 4. Technical Guidance

**Conventions for this module:**
- Test project: `ProductivityPlatform.UnitTests`
- Naming: `{ValidatorName}Tests` per validator
- Pattern: Arrange → create validator + command/query with test data; Act → `validator.Validate(command)`; Assert → check `result.IsValid`, error messages, error property names
- Cover: empty/null fields, max length exceeded, invalid enum values, future date rules, email format, password complexity
- Target files by module folder under `Application/`: `Auth/`, `Tasks/`, `Projects/`, `Notes/`, `Calendar/`, `Dashboard/`
- Use xUnit `[Theory]` with `[InlineData]` for multiple test cases per validation rule

**Concepts practiced in this task:**
- Systematic validator testing
- Theory-based test data patterns
- Edge case enumeration for validation rules
- Coverage-driven test design
